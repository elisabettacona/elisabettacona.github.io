About

-

The main goal of this project was to review the user experience and the user interface of ATM's web application and kiosks in order to enhance its usability.

Our starting point was a research about ATM transportation in Milan and how people uses the application and the kiosks. Using this methodology we defined user needs and problems which we took advantage of in order to fix usability issues.

With interviews, observations and surveys the research pointed out how difficult was for people to use the kiosks because of its unintuitive interface and how people preferred to buy tickets at the Tabacchi and directly ask to the tabacconist which one was the correct ticket to buy.

Actually the interface wasn't matching at all user mental models and there was also lots of scopes for errors.


Collecting all the information were designed 4 personas and 4 main tasks to help us define the final structure and once complete, start with user testing.

Also, after research and card sorting results, an information architecture was built.

To create the project we used Balsamiq first, Sketch and Invision then. Here the Invision prototype available.

Web App

ATM Milano

For each task defined a user flow was created. Below the example of the most common task "buy a single urban ticket". User can access the functionality directly on the fast services section and buy the ticket in an easy way.

Starting with the kiosk, low fidelity wireframe were created and tested in order to check the usability and enhance it after test result collection and analisys.

After user testing some changes on wireframes were done because users found difficulties when trying to buy a cumulative ticket. For this reason the screen was split in two in order to let the user focus on one thing at a time and not overload him.


Above the starting screen with ticket type and zone selection in one page. Below the same information split in two different steps.

Using all the feedback received, an high fidelity prototype were built for the Web App, always based on the main tasks identified.

Again, once this step was complete, another user test were run and with the results in mind the layout was modified accordingly.
